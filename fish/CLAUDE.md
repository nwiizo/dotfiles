# Fish Configuration Notes

Fish is managed entirely by Home Manager. **The source of truth is
[`../home/fish.nix`](../home/fish.nix)** — `~/.config/fish/config.fish` is
generated from it on `home-manager switch`.

## Where to edit what

| Change | Location |
|---|---|
| Shell init (env, PATH, XDG, Homebrew) | `home/fish.nix` → `shellInit` |
| Interactive setup (FZF, prompt vars, key bindings) | `home/fish.nix` → `interactiveShellInit` |
| Abbreviations (`g`, `gst`, `k`, ...) | `home/fish.nix` → `shellAbbrs` |
| Inline wrappers (`ls`, `cat`, `grep`, ...) | `home/fish.nix` → `functions` |
| Larger functions (`fish_prompt`, `ai_*`, `jj_*`, `update_all`, ...) | `fish/functions/*.fish` (referenced via `xdg.configFile`) |
| Sponge fish 4.x compat patch | `fish/conf.d/zz_sponge_compat.fish` |
| Plugin list | `home/fish.nix` → `plugins`; non-nixpkgs sources are flake inputs in `flake.nix` |

## Apply changes

```bash
home-manager switch --flake .#nwiizo
exec fish   # reload current shell if needed
```

`update_all` runs `home-manager switch` plus tool updates in one go.

## Don't do

- Don't edit `~/.config/fish/config.fish` directly — it's a symlink into
  `/nix/store` and gets overwritten.
- Don't add Fisher plugins via `fisher install`. Add the plugin to
  `home/fish.nix` (`programs.fish.plugins`); fetch from `pkgs.fishPlugins.*`
  if it's in nixpkgs, or as a flake input otherwise.
- Don't define `fish_prompt` / `fish_right_prompt` anywhere except
  `fish/functions/`. They are *not* generated by Starship anymore.
- Don't reintroduce `conf.d/atuin.fish` / `direnv.fish` / `zoxide.fish` —
  Home Manager generates these via `programs.{atuin,direnv,zoxide}.enableFishIntegration`.
